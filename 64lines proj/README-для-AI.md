# Инструкция для ИИ-агентов по работе с базой знаний (Vault)

Привет! Если ты нейросеть (ИИ-агент), которая читает эту папку в целях анализа проекта, рефакторинга или поиска контекста, пожалуйста, следуй этим правилам для быстрого и точного поиска информации.

---

## 📂 Структура каталогов базы знаний

Вся информация о шахматной школе распределена по строго типизированным папкам:
1. `00-Index/` — Индексы и карты связей. Начинай анализ отсюда.
2. `01-Обзор-проекта/` — Назначение системы, роли пользователей, стек технологий и конфигурации.
3. `02-База-данных/` — Схема Prisma и подробные описания каждой модели PostgreSQL.
4. `03-API/` — Эндпоинты, сгруппированные по сущностям. Описание параметров запросов и ответов.
5. `04-Компоненты/` — React-компоненты интерфейса (входные пропсы, вызываемые API).
6. `05-Страницы-и-маршруты/` — Карты переходов по страницам Next.js.
7. `06-Бизнес-процессы/` — Сценарные описания (флоу), сшивающие UI, API и БД в единую цепочку.
8. `07-Интеграции-и-инфраструктура/` — Внешние системы (NextAuth, YooMoney, Jitsi, Nodemailer).
9. `08-Скрипты-и-утилиты/` — CLI скрипты миграций, администрирования и тестов.
10. `99-Глоссарий/` — Шахматные термины (FEN, PGN, Elo) и технические понятия.

---

## 🎯 Правила навигации и поиска контекста

1. **Точка входа**: Всегда начинай поиск с [[00-Главная|Главной страницы (MOC)]].
2. **Поиск сущностей по имени файла**:
   - Каждая сущность бэкенда (модель БД) описана в файле формата `Model-[Название].md` в папке `02-База-данных/`.
   - Ключевые компоненты названы точно так же, как в коде: например, [[teacher-hub]] или [[Puzzles]].
3. **Использование вики-ссылок (Wikilinks)**:
   - Все связи в базе реализованы через вики-ссылки формата `[[Model-User]]`.
   - Если ты анализируешь файл и видишь упоминание другой сущности, которая связана с текущей — ты найдешь её по ссылке в тексте. Пользуйся этим для построения пути выполнения (Execution Path) без лишнего перебора всех файлов.
4. **Сценарный анализ**:
   - Если тебе нужно понять, как работает конкретная фича (например, покупка подписки или решение задач), не читай код разрозненно. Начни с папки `06-Бизнес-процессы/` (например, [[Покупка-курса-модуля]]), где описан весь пошаговый сценарий от кнопки на фронтенде до строки в БД.

## 🧬 Соответствие кода и базы знаний
В каждой технической заметке в разделе **"Расположение в коде"** указан точный относительный путь к файлу в исходниках проекта. Это позволяет сопоставить теоретическое описание из Obsidian с реальным кодом.
