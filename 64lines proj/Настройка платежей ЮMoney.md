# Настройка платежей ЮMoney (Quickpay Webhook)

Этот документ содержит пошаговую инструкцию по настройке автоматического приёма платежей через ЮMoney для администратора шахматной школы "64 линии".

---

## Чек-лист для настройки в личном кабинете ЮMoney

Для того чтобы платежи автоматически зачислялись на вашем сайте, вам необходимо настроить HTTP-уведомления в вашем кошельке ЮMoney:

1. **Войдите в ЮMoney** под своей учетной записью.
2. Перейдите в **Профиль** → **Настройки платежей** → **HTTP-уведомления** (или по прямой ссылке [yoomoney.ru/transfer/mysettings/http-notification](https://yoomoney.ru/transfer/mysettings/http-notification)).
3. Введите **Адрес для уведомлений (Webhook URL)**:
   - В проде (на Vercel): `https://your-site-domain.vercel.app/api/yoomoney/callback`
   - Для тестирования локально (с использованием ngrok): `https://your-ngrok-subdomain.ngrok-free.app/api/yoomoney/callback`
4. Нажмите **"Показать секрет"** и скопируйте предоставленное **Секретное слово** (Notification Secret).
5. Поставьте галочку **"Отправлять уведомления"** и сохраните изменения.

---

## Настройка переменных окружения

Полученное секретное слово необходимо добавить в настройки проекта:

### Локальная разработка (.env)
Добавьте в файл `.env` в корне проекта строчку:
```env
YOOMONEY_SECRET="ВАШЕ_СЕКРЕТНОЕ_СЛОВО_ИЗ_ЮМОNEY"
```

### Настройка на хостинге (Vercel)
1. Откройте панель управления **Vercel** вашего проекта.
2. Перейдите во вкладку **Settings** → **Environment Variables**.
3. Добавьте новую переменную:
   - **Key**: `YOOMONEY_SECRET`
   - **Value**: `ВАШЕ_СЕКРЕТНОЕ_СЛОВО_ИЗ_ЮМОNEY`
4. Нажмите **Save** и перезапустите деплой (Redeploy).

> [!WARNING]
> Без настройки переменной `YOOMONEY_SECRET` автоматические платежи ЮMoney подтверждаться НЕ БУДУТ! Заказы будут зависать в статусе `PENDING` (Ожидание), а администратор увидит предупреждение в панели "Продажи".

## Тестовый сценарий локальной проверки (Mock Webhook)

Вы можете протестировать обработку платежа вебхуком локально с помощью скрипта. Мы подготовили тестовый скрипт, который генерирует корректный SHA1-хэш подписи с вашим секретом и отправляет его на локальный обработчик.

### Шаг 1: Создайте тестовый заказ в БД
Сначала создайте заказ (Purchase) на вашем сайте, выбрав оплату ЮMoney. Получите его ID (например, `cl...` или другой UUID).

### Шаг 2: Запустите локальный скрипт тестирования
В корне проекта выполните:
```bash
node scripts/test-yoomoney-webhook.js <PURCHASE_ID> [AMOUNT]
```
*(где `<PURCHASE_ID>` — ID созданного заказа, а `[AMOUNT]` — необязательная сумма платежа, по умолчанию 309 ₽ с учётом комиссии за курс в 300 ₽)*

Скрипт сам прочитает `YOOMONEY_SECRET` из вашего локального `.env`, подпишет запрос и отправит POST на `http://localhost:3000/api/yoomoney/callback`.

### Шаг 3: Проверка в панели администратора
Зайдите в Панель тренера → раздел "Продажи". Заказ должен автоматически сменить статус на "Оплачено", а выручка — увеличиться на указанную сумму.
