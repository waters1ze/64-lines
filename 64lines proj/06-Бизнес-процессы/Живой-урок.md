# Бизнес-процесс: Проведение Живого Урока (Live Session)

Интерактивный живой урок позволяет тренеру проводить индивидуальные или групповые занятия прямо на платформе.

---

## 🏃 Процесс инициализации и синхронизации урока

```mermaid
sequenceDiagram
    autonumber
    actor Тренер
    actor Ученик
    participant Сервер as Next.js API
    participant БД as PostgreSQL (Prisma)

    Тренер->>Сервер: Нажимает "Начать урок с Учеником X" (POST /api/live)
    Сервер->>БД: Создает LiveSession (status: "ACTIVE", jitsiRoomName: "room-XYZ")
    Сервер-->>Тренер: Возвращает ID сессии и имя комнаты
    Note over Тренер: В iframe загружается Jitsi-звонок в комнате "room-XYZ"
    
    Ученик->>Сервер: Опрашивает API (GET /api/live)
    Сервер-->>Ученик: Возвращает активную сессию (room-XYZ)
    Note over Ученик: У ученика автоматически всплывает окно входа в урок
    Ученик->>Ученик: Подключается к Jitsi-звонку

    %% Синхронизация ходов
    Тренер->>Тренер: Делает ход "e4" на демонстрационной доске
    Тренер->>Сервер: Отправляет PUT /api/live { currentFen: "rnbqkbnr/pppppppp/...", pgn: "1. e4" }
    Сервер->>БД: Обновляет LiveSession

    rect rgb(240, 240, 240)
        Note over Ученик: Таймер опроса (раз в 1.5 секунды)
        Ученик->>Сервер: Отправляет GET /api/live
        Сервер-->>Ученик: Возвращает { currentFen: "rnbqkbnr/pppppppp/...", pgn: "1. e4" }
        Ученик->>Ученик: Доска автоматически перерисовывает ход "e4"
    end

    Тренер->>Сервер: Нажимает "Завершить урок" (PUT /api/live { status: "ENDED" })
    Сервер->>БД: Обновляет LiveSession
    Ученик->>Ученик: Уведомление "Урок завершен", отключение от Jitsi
```

---

## ⚙️ Особенности Jitsi и доски
- Видеосвязь вставляется через официальный скрипт `https://8x8.vc/external_api.js` (бесплатный публичный сервер Jitsi Meet).
- Для стабильности шахматная доска не передает видеопоток, а шлет только текстовые строки FEN. Это гарантирует бесперебойную работу доски даже на медленном 3G-интернете.
