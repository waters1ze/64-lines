# Бизнес-процесс: Решение тактических задач

Интерактивный тренажер тактики автоматически подбирает шахматные задачи под уровень игры ученика и отслеживает рост его шахматной силы по рейтингу Elo.

---

## 🏃 Алгоритм подбора и фиксации задачи

```mermaid
flowchart TD
    Start["Ученик заходит в раздел Задачи"] --> CheckLimit{"Пользователь Premium?"}
    
    CheckLimit -- "Да" --> GetRating["Получение Elo рейтинга пользователя из БД"]
    CheckLimit -- "Нет" --> CheckSolved{"Решено задач сегодня >= 5?"}
    
    CheckSolved -- "Да" --> LimitBlocked["Блокировка экрана, предложение купить Premium"]
    CheckSolved -- "Нет" --> GetRating
    
    GetRating --> FetchDb["Запрос к таблице Puzzle (исключая решенные SolvedPuzzle)"]
    FetchDb --> FilterRating["Фильтр задач в диапазоне User.rating +/- 100"]
    FilterRating --> PuzzleFound{"Найдена задача?"}
    
    PuzzleFound -- "Да" --> Render["Рендеринг шахматной доски (Puzzles.tsx)"]
    PuzzleFound -- "Нет" --> ExpandRange["Расширение диапазона до +/- 300"]
    ExpandRange --> Render
    
    Render --> Play["Пользователь решает задачу на доске"]
    Play --> Submit["Отправка POST /api/puzzles/submit { isCorrect }"]
    Submit --> EloCalc["Пересчет Elo рейтинга пользователя (+/- в зависимости от результата)"]
    EloCalc --> SaveResult["Сохранение в SolvedPuzzle, инкремент puzzlesSolvedToday"]
    SaveResult --> StreakUpdate["Обновление дней активности (activityStreak)"]
    StreakUpdate --> ShowChanges["Вывод изменения рейтинга (+15 / -12) на экран"]
```

---

## 📈 Elo рейтинг
- Используется классическая формула шахматного рейтинга с коэффициентом K=32.
- Шанс на победу (математическое ожидание E) рассчитывается как:
  $$E = rac{1}{1 + 10^{(R_{puzzle} - R_{user}) / 400}}$$
- Изменение рейтинга пользователя:
  $$Delta R = K cdot (S - E)$$
  где $S = 1$ при правильном решении, и $S = 0$ при ошибке.
