## Стек технологий
- **Фреймворк**: Next.js 14 (App Router)
- **Язык**: TypeScript
- **База данных**: PostgreSQL (через Prisma ORM)
- **Аутентификация**: NextAuth.js
- **Стилизация**: Tailwind CSS, shadcn/ui, Lucide React (иконки)
- **Шахматная логика**: `chess.js` (правила и генерация ходов) + `react-chessboard` (UI доски)

---

## Структура папок
Корневая структура проекта:
- `app/` — основные маршруты (App Router). Включает API роуты (`app/api/`) и страницы (`app/(pages)/` или корень).
- `components/` — React компоненты. Включает UI-элементы (`ui/`) и бизнес-компоненты (например, `teacher-hub.tsx`, `Puzzles.tsx`).
- `lib/` — утилиты и конфигурация (например, инициализация Prisma клиента `db.ts`).
- `prisma/` — схема базы данных (`schema.prisma`).
- `scripts/` — скрипты для административных задач, парсинга PGN, миграций и прочего.

## Роли и уровни доступа
Система поддерживает три основные роли (хранятся в `Role` enum базы данных):

1. **ADMIN (Администратор)**
   - Имеет полный доступ ко всем функциям.
   - Может управлять курсами, пользователями, товарами.
2. **TEACHER (Учитель / Тренер)**
   - Имеет доступ к `Teacher Hub` (Кабинет тренера).
   - Может приглашать учеников, назначать им домашние задания (`Homework`).
   - Имеет возможность проводить живые уроки (Live Sessions).
3. **STUDENT (Ученик / Покупатель)**
   - Может решать тактические задачи (Puzzles).
   - Может привязываться к учителю и выполнять домашние задания.
   - Имеет доступ к купленным курсам и видео.
   - Отслеживает свой прогресс и рейтинг.

## Ключевые компоненты

### `teacher-hub.tsx` (Кабинет Тренера)
Самый объемный компонент, являющийся ядром для взаимодействия тренера и учеников. 
Включает в себя:
- Раздел домашних заданий (создание, назначение, проверка с помощью шахматной доски).
- Список учеников и прогресс.
- Живые уроки (с интеграцией Jitsi и синхронизированной шахматной доской).
- Управление контентом тренера.

### `Puzzles.tsx` (Тактические задачи)
Компонент для решения пользователями тактических задач. Загружает FEN позиции с сервера, обрабатывает ходы ученика и проверяет их на соответствие правильному решению FEN (через API). Включает систему рейтинга Elo.

### `LiveLessonBoard.tsx`
Синхронизированная шахматная доска для живых занятий, позволяющая тренеру и ученику видеть одни и те же ходы в реальном времени.

## Схема базы данных (Prisma)

Основные сущности и их связи:

### `User` (Пользователь)
Сердце системы. Связан практически со всеми другими таблицами.
- У учеников может быть привязан тренер: `teacherId`.
- У тренера есть массив `students`.

### `Homework` (Домашнее задание)
Задания, назначаемые тренером ученику.
- Содержит PGN партии/задачи.
- Отслеживает `progress`, `solved`, и `attempts` (попытки решения).

### `Course` и `Module` (Обучающие материалы)
- Курсы содержат видео или PGN. Могут быть куплены (сущность `Purchase`).
- Доступ к курсам проверяется через `isPremium` статус пользователя или индивидуальные покупки `Purchase`.

### `Puzzle` (Задачи)
Хранит огромную базу шахматных задач.
- `fen` — начальная позиция задачи.
- `moves` — строка правильных ходов (SAN).
- `rating` — рейтинг задачи.

### `LiveSession`
Управляет текущими онлайн-уроками.
- Хранит ID учителя и ученика.
- Содержит URL комнаты для видеозвонка и текущее состояние доски (`currentFen`).

### `Message` и `Notification`
Внутренняя система чата и уведомлений для связи тренеров с учениками.
