# NextAuth.js
> Библиотека аутентификации для Next.js, отвечающая за генерацию сессий, валидацию паролей и хранение ролей доступа.

## Расположение в коде
`app/api/auth/[...nextauth]/route.ts`

## Детали
- **Провайдер**: `CredentialsProvider` (вход по Email и Паролю).
- **Тип сессии**: Зашифрованный JWT (JSON Web Token), хранящийся в httpOnly-куках браузера.
- **Время жизни сессии**: По умолчанию 30 дней.
- При авторизации NextAuth считывает данные из таблицы [[Model-User]] через Prisma:
  - Сверяет переданный пароль с хэшем `passwordHash` с помощью библиотеки `bcrypt`.
  - Записывает роль (`role`), ID пользователя (`id`) и статус Premium (`isPremium`) в JWT-сессию.

## Связи
- Использует: [[Model-User]]
- Используется в: [[providers]], [[Главная-page]], [[API-Auth]], [[00-Индекс-API|Все API маршруты]]

## Заметки / особенности
NextAuth сессия является статической до момента обновления токена. Если пользователь оплачивает подписку в процессе работы, для предотвращения отображения устаревших данных из JWT, бэкенд на странице [[Главная-page]] делает повторный прямой запрос в базу данных и переопределяет статус на клиенте.
