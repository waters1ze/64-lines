# Cron очистка базы данных
> Регулярная автоматическая задача по очистке старых данных для снижения нагрузки на PostgreSQL и соблюдения политики конфиденциальности.

## Расположение в коде
`app/api/cron/cleanup-messages/route.ts`

## Детали
- Выполняется раз в 24 часа.
- Вызывает эндпоинт `/api/cron/cleanup-messages`.
- Ищет в БД записи в таблице [[Model-Message]], созданные более 30 дней назад.
- Физически удаляет (deleteMany) сообщения только в том случае, если оба участника чата (отправитель и получатель) имеют Premium-статус в положении `false`. Если хотя бы у одного пользователя есть Premium, история диалога сохраняется.

## Связи
- Использует: [[Model-Message]], [[Model-User]]
- Используется в: [[API-Cron]]

## Заметки / особенности
Для автоматического запуска задачи в продакшн среде (например, на Vercel) в корне проекта настраивается конфигурационный файл `vercel.json`, где прописывается расписание вызова эндпоинта:
```json
{
  "crons": [
    {
      "path": "/api/cron/cleanup-messages",
      "schedule": "0 0 * * *"
    }
  ]
}
```
