# Страница Главная (Root Page)
> Главная страница приложения. Она проверяет сессию пользователя на стороне сервера, загружает стартовые данные из базы данных PostgreSQL через Prisma и передает их в клиентский компонент личного кабинета.

## Расположение в коде
`app/page.tsx`

## Детали
### Загружаемые данные из БД (Server-Side):
- Профиль текущего пользователя (рейтинг Elo, Premium статус, streak активности).
- Список курсов (`db.course`), видеозаписей (`db.video`) и дебютных баз (`db.opening`).
- Если роль Ученик: список домашних заданий ученика, история его покупок.
- Если роль Учитель: список привязанных учеников, назначенные ДЗ, история всех продаж школы.

### Логика проверок (Middlewares):
- Если у пользователя истек срок Premium-подписки (`premiumUntil < new Date()`), сервер автоматически сбрасывает статус `isPremium = false` в базе данных перед рендерингом.
- Если сессия авторизована, но пользователя нет в БД, принудительно перенаправляет на разлогин `/api/auth/signout`.
- При открытии страницы тренером или учеником происходит автоматическая каскадная очистка просроченных домашних заданий (`dueDate < now()`).
